## Color Scheme

### Top Bar
- Background: Black (0, 0, 0, 0.8)
- Dialog: Dark blue-gray (0.1, 0.1, 0.15, 0.95)

### Panels
- Attacker: Blue tint (0.2, 0.3, 0.5, 0.5)
- Defender: Red tint (0.5, 0.2, 0.2, 0.5)

### Text Colors
- Title: Bright red (1.0, 0.2, 0.2)
- Attacker name: Light blue (0.5, 0.8, 1.0)
- Defender name: Light red (1.0, 0.5, 0.5)
- Results: Yellow (1.0, 1.0, 0.3)
- Combat continues: Orange-yellow (1.0, 0.8, 0.2)

### Dice
- Box: White (1.0, 1.0, 1.0, 0.9)
- Dots: Black (0, 0, 0)

### Combat Indicator
- Color: Red (1.0, 0, 0)
- Alpha: Pulses 0.4 to 0.8
- Material: Transparent standard shader

## User Flow

1. **Combat Starts**
   - Top bar slides in from top
   - Red indicator appears at combat hex
   - Dice boxes appear with random faces

2. **Rolling Animation (1 second)**
   - Dice rapidly cycle through dot patterns
   - Dice containers shake left/right and up/down
   - Red indicator pulses at combat location

3. **Results Shown**
   - Dice settle on final values
   - Result text appears (who wins, damage dealt)
   - Continue button becomes clickable

4. **Player Clicks Continue**
   - Top bar remains (for subsequent combat rounds or fades out)
   - Red indicator disappears
   - Game resumes

5. **Multi-Round Combat**
   - Title updates: "COMBAT - Round 2"
   - Indicator stays visible
   - Process repeats for each round
